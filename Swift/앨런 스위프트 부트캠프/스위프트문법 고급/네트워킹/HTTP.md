# API ====================================================================
API = application programming interface = 서버와 통신하기 위한 어떤계약이나 룰 , 사용법 , 약속

요청메세지 = 리퀘스트 = query
응답메세지 = 리스폰 = data

https://www.google.com:443/search?q=swift&hl=ko
여기서

445/ = 포트
포트는 아파트의 호수 같은거, 서버가 동작할때 실제 주소에 여러가지앱이 실행될수도있기때문에 그런것들을 구분하기위해서 사용 / 많이생략되기도함

?q=swift&hl=ko = 쿼리파라미터 
쿼리 파라미터 : 딕셔너리처럼 key = value의 형태이며 ?로시작하고 &로 추가가능
쿼리파라미터를 보통 GET메서드에서는 주소에 쓰고 https://www.google.com:443/search?q=swift&hl=ko 이런식으로
일반적으로 GET메서드에서는 쿼리파라미터로 데이터를 전송한다. (주소에붙히는거) 예: 검색어 / 정렬기준

# Rest API ====================================================================
요청방식에 대한 약속 = REST한 형식의 Api로 하자
http://(사이트주소)/movielists ===> 상영중 영화목록 요청 GET
http://(사이트주소)/movielists?open=tomorrow ===> 내일 오픈예정 영화 목록 GET
http://(사이트주소)/movie-reservation ===> 영화예약 POST
