1. Copy On Write는 어떤 방식으로 동작하는지 설명하시오.

COW란 프로그래밍에서 데이터를 복사시에 실제로 복사되지않고 동일한 값을 참조하다가 데이터 변경이 발생될시에
값을 복사해 값을 변경하는 기법입니다.

이 기법은 스위프트에서는 컬렉션타입을 복사해서 사용할때 일어나게됩니다.

var array1: [Int] = [0,1,2,3]
var array2 = array1

지금 이상황에서는 array1을 생성하고 array2에 넣은 상황입니다.
COW때문에 값이 복사되어 array2에 들어가는것이 아니라
array1과 array2는 동일한 메모리 공간을 가리키고 있게됩니다.
그리고 원본이나 복사본의 값이 변경이 일어날때에만 값이 복사가 되는것입니다.
array1.append array2.append 같이 값이 수정되면 그때
array2의 값이 메모리에 할당되는겁니다.
더이상 같은 메모리 공간을 가리키고 있는게 아니죠

이런 cow를 하는 이유는 실제 복사를 하더라도 수정을 하지않거나 앞으로도 안하는경구가 있는데
실제 수정이 이루어질때 복사를하고 그전에 참조를 통해 불필요한 복사를 줄이자는게
COW의 사용 이유입니다.

13 Delegates와 Notification 방식의 차이점에 대해 설명하시오.

notification 방식은 이벤트를 1:N으로 전달할 때 용이합니다. NotificationCenter라는 싱글톤객체를 기반으로 이벤트 발생여부를 옵저버를 등록한 객체에게 전달하는 방식으로 구성됩니다. 따라서 다수의 객체에게 손쉽게 이벤트 전달이 가능합니다. 하지만 제 3 객체를 필수적으로 필요로 하며, key값으로 Notification의 이름을 사용하기 때문에 컴파일 중 구독이 제대로 이루어져있는지 확인할 수 없다는 단점이 존재합니다.
