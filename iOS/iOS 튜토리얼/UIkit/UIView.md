```swift
뷰는 자신의영역에서  데이터를 표시하고 터치이벤트를 처리한다.
여기에 필요한 기본적인 기능들은 UIKit 프레임워크에 구현되어있다.
UIView는 모든 뷰가 공통적으로 상속받는 클래스고 여기에는 UI구현에 필요한 필수 기능들이 구현되어있다.

화면을 그리는 작업은 고비용의 작업 -> cpu와 메모리를 많이 사용
view 이벤트가 발생할때마다 화면을 다시 그리는데 매번 다시그리면 전체적인 성능이 하락함
그렇기 때문에 view는 비트맵 캐시를 사용한다
컨텐트를 그릴때마다 최종결과를 캐시에 저장해두고 바뀔때마다 새로 그리지않고
최종 결과 캐시에서 바뀐부분만 다시 그린다 이것을 
on demand drawing model 이라고 한다.

뷰는 자신의 프레임에 포함되어있는 다른 뷰를 관리한다
이때 계층구조로 관리하는데
계층구조에서 위에있는 뷰를 슈퍼뷰 아래있는 뷰를 서브뷰라고한다
뷰는 하나이상의 서브뷰를 가질수있다.
그런데 슈버퓨는 항상 하나이다.
뷰는 서브뷰를 배열로 가진다. 
서브뷰가 배열에 저장된 순서는 화면에 표시되는 순서이다.
배열 가장 마지막에 표시된 서브뷰가 화면에선 제일 위에 표시된다.
서브뷰는 슈퍼뷰의 속성을 상속받고 영향을 받는다
예를들어 슈퍼뷰의 위치와 크기가 업데이트되면 서브뷰도 이에 맞춰 업데이트된다.



뷰의 위치와 크기를 프레임 이라고 한다
타입은 CGRect 구조체 이다. 그 안엔 이렇게 있다.

struct CGRect {
    var origin: CGPoint
    var size: CGSize
}

struct CGPoint {
    var x: CGFloat
    var y: CGFloat
}

struct CGSize {
    var width: CGFloat
    var height: CGFloat
}

var frame: CGRect


코드로 뷰를 추가하는 방법

let fream = CGRect(x: 50, y: 50, width: 100, height: 100)
let v = UIview(fream: fream)
view.addSubview(v)

코드로 뷰를 만들때는 
프레임을 만들고 > 뷰를만들고 > 뷰를 올린다
이렇게 한뒤 제약을 추가한다. ( 오토레이 아웃 )




뷰의 그리기 성능의 관련된 속성
투명도
슈퍼뷰에서 알파속성을 건드리면 서브뷰에도 영향을 준다.


프레임
clips to bounds 옵션을 선택하면 프레임을 벗어난 부분은 표시되지않는다.



opaque 옵션
투명도 설정을하면 겹쳐지는부분의 색을 합성하는 과정이 필요한데 이건 cpu의 성능을 많이쓴다 ( 물론 체감 불가)
opaque속성을 클릭하면 뷰를 화면에 출력할때 어떻게 최적하 할지 결정한다.
애플의 가이드 라인은
알파값이 1.0 인 경우에는 opaque속성을 true로
1.0 미만인경우에는 opaque속성을 false 로 하면 된다.

clears graphics context 
이 속성이 체크되어있으면 뷰가 새로 그려질때마다 이전의 내용을 전부 삭제하고 다시그리게 된다.
이 속성을 체그하지않으면 그리기 성능은 향상하지만 구현에 따라서 이전내용이 표시될수 있기때문에 문제가 발생할수도 있다.





새로운 뷰를 만드는 방법
뷰를 만들려면 위치와 크기가 필요하다.
위치와 크기를 우리는 프레임이라고 표현한다.
그리고 프레임은 항상 슈퍼뷰의 좌표내에서 뷰의 위치와 크기를 표현한다.
    override func viewDidLoad() {
        super.viewDidLoad()
         
        let ftrame = CGRect(x: 50, y: 50, width: 100, height: 100) //프레임을 만들고
        let v = UIView(frame: ftrame) //뷰 생성자에 넣는다
        view.addSubview(v) //그리고 서브뷰에 만든 뷰를 추가한다
        
        //프레임을 만들고 > 뷰를만들고 > 뷰계층에 추가한다 > 여기에 제약을 추가
        
    }


뷰의 배경색을 바꾸고 싶다면
v.backgroundColor = .systemBlue






뷰를 아웃렛으로 연결하지않고 태그라는 방식으로 뷰의 속성은 건드릴수 있는데.
보통 잘 사용하지는 않는다
기본적으로 설정해주기 전까지는 모든 뷰의 태그는 0이다.

if let v = view.WithTag(0) {
    v.backgroundColor = .black
}



인터렉션 항목에 두가지 옵션이 존재하는데
한개는 User Interaction Enabled 이다. 이것은 뷰가 프레임 내부에서발생하는 터치이벤트를 처리할거냐는 옵션이고 기본적으로 체크되어있다.

하나는 Multiple touch 이다
멀티 터치 즉 다중 터치를 지원할것이냐는 옵션이다.
```
